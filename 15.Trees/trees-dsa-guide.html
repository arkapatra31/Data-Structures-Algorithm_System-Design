What is a tree?

A tree is a hierarchical data structure of nodes connected by edges. Unlike arrays or linked lists, it branches out. The key rule: every node has exactly one parent, except the root (which has none), and there are no cycles.

Key terminology

Node representation

The simplest tree is a binary tree — each node has at most 2 children. A node holds a value and two pointers (left, right). Everything else builds on top of this.

Tree traversals

Traversal = visiting every node in a specific order. The DFS ones only differ in when you process the current node relative to its children. The name tells you where "Root" goes.

Key insight

Inorder on a BST gives sorted output. Preorder is used for serialization. Postorder for deletion. Level order is BFS with a queue. Know both recursive and iterative versions for interviews.

Binary search tree (BST)

A BST enforces one rule: for every node, left subtree values < node < right subtree values. This lets you search in O(log n) by eliminating half the tree at each step.

Operations

Insert — walk left/right until you find an empty spot. Search — same walk, return when found or null. Delete — the tricky one: 0 children (just remove), 1 child (replace with child), 2 children (replace with inorder successor).

Complexity

OperationAverageWorst (skewed)
SearchO(log n)O(n)
InsertO(log n)O(n)

AVL trees (self-balancing BST)

A BST degenerates to O(n) when you insert sorted values (it becomes a linked list). AVL trees guarantee O(log n) by ensuring the height difference between left and right subtrees is at most 1. When violated, a rotation rebalances.

How it works

After every insert or delete, walk back up to root updating heights. At each node, check balance factor = height(left) − height(right). If |balance| > 1, apply the appropriate rotation. The rotation case depends on where the imbalance originated (left-left, right-right, left-right, or right-left).

Trie (prefix tree)

A trie stores strings character by character along branches. Shared prefixes share the same path — cat and car share c→a. Search and insert are both O(L) where L is word length.

Heap / Priority queue

A heap is a complete binary tree where every parent is greater (max-heap) or smaller (min-heap) than its children. The beauty: it's stored as a flat array — no pointers needed.

Array representation: [50, 30, 40, 10, 20, 35, 15]
For index i: parent = (i-1)//2, left child = 2*i+1, right child = 2*i+2

Key operations

Segment tree

A segment tree answers range queries (sum, min, max over a subarray) in O(log n) while supporting point updates in O(log n). Each node stores the answer for a range, and its children split that range in half.

Array: [3, 5, 7, 4, 8, 9]
Each leaf = array element. Each parent = sum of its children. Query sum(1..4) = 5+7+4+8 = 24 in O(log n).

Three variants

Range update + range query
