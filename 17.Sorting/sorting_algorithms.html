Bubble Sort O(n²)

Repeatedly steps through the list, compares adjacent elements, and swaps them if they are in the wrong order. Each complete pass "bubbles" the largest unsorted element to its correct position at the end.

How it works

  1. Start from index 0, compare each pair of adjacent elements.
  2. If the left element is greater than the right, swap them.
  3. After one full pass, the largest element is at the end.
  4. Repeat for the remaining unsorted portion.
  5. Optimisation: if no swaps in a pass, exit early.
Best
O(n)
Average
O(n²)
Worst
O(n²)
Space
O(1)
StableIn-placeAdaptive (early exit)

Visual walkthrough — Pass 1

def bubble_sort(arr):
    a = arr[:]
    n = len(a)
    for i in range(n):
        swapped = False
        for j in range(n - 1 - i):
            if a[j] > a[j + 1]:
                a[j], a[j + 1] = a[j + 1], a[j]
                swapped = True
        if not swapped:  # early exit
            break
    return a

arr = [38, 27, 43, 3, 9, 82, 10]
print(bubble_sort(arr))  # [3, 9, 10, 27, 38, 43, 82]

Selection Sort O(n²)

Divides the array into a sorted and unsorted region. Repeatedly finds the minimum element from the unsorted region and swaps it into the next position in the sorted region.

How it works

  1. Start with the entire array as unsorted.
  2. Scan the unsorted region to find the minimum element.
  3. Swap the minimum with the first unsorted element.
  4. The sorted region grows by one. Repeat.
Best
O(n²)
Average
O(n²)
Worst
O(n²)
Space
O(1)
Not stableIn-placeFewest swaps: O(n)

Visual walkthrough — Pass 1

def selection_sort(arr):
    a = arr[:]
    n = len(a)
    for i in range(n):
        min_idx = i
        for j in range(i + 1, n):
            if a[j] < a[min_idx]:
                min_idx = j
        a[i], a[min_idx] = a[min_idx], a[i]
    return a

arr = [38, 27, 43, 3, 9, 82, 10]
print(selection_sort(arr))  # [3, 9, 10, 27, 38, 43, 82]

Insertion Sort O(n²)

Builds the final sorted array one element at a time. Picks each element and inserts it into its correct position among the already-sorted elements — like sorting playing cards in your hand.

How it works

  1. Consider the first element as already sorted.
  2. Pick the next element (the "key").
  3. Compare the key with sorted elements, shift larger ones right.
  4. Insert the key into the gap. Repeat for all elements.
Best
O(n)
Average
O(n²)
Worst
O(n²)
Space
O(1)
StableIn-placeAdaptiveBest for nearly-sorted data

Visual walkthrough — inserting 27

def insertion_sort(arr):
    a = arr[:]
    for i in range(1, len(a)):
        key = a[i]          # element to insert
        j = i - 1
        while j >= 0 and a[j] > key:
            a[j + 1] = a[j]  # shift right
            j -= 1
        a[j + 1] = key      # insert key
    return a

arr = [38, 27, 43, 3, 9, 82, 10]
print(insertion_sort(arr))  # [3, 9, 10, 27, 38, 43, 82]

Merge Sort O(n log n)

A divide-and-conquer algorithm that recursively splits the array into halves until each sub-array has one element, then merges them back together in sorted order.

How it works

  1. Divide the array into two halves at the midpoint.
  2. Recursively sort each half (base case: single element).
  3. Merge the two sorted halves by comparing front elements.
  4. Continue until all elements are merged into one sorted array.
Best
O(n log n)
Average
O(n log n)
Worst
O(n log n)
Space
O(n)
StableNot in-placeGuaranteed O(n log n)

Visual — Divide and merge

def merge_sort(arr):
    if len(arr) <= 1: return arr[:]
    mid = len(arr) // 2
    left = merge_sort(arr[:mid])
    right = merge_sort(arr[mid:])
    return _merge(left, right)

def _merge(left, right):
    result, i, j = [], 0, 0
    while i < len(left) and j < len(right):
        if left[i] <= right[j]:
            result.append(left[i]); i += 1
        else:
            result.append(right[j]); j += 1
    result.extend(left[i:])
    result.extend(right[j:])
    return result

arr = [38, 27, 43, 3, 9, 82, 10]
print(merge_sort(arr))  # [3, 9, 10, 27, 38, 43, 82]

Quick Sort O(n log n)

Selects a "pivot", partitions the array so elements smaller go left and larger go right, then recursively sorts the two partitions.

How it works (Lomuto partition)

  1. Choose a pivot (commonly the last element).
  2. Maintain boundary pointer i for the "less than pivot" region.
  3. Scan left to right: if element < pivot, swap to position i, advance i.
  4. Swap pivot into position i — it is now in its final position.
  5. Recurse on left and right partitions.
Best
O(n log n)
Average
O(n log n)
Worst
O(n²)
Space
O(log n)
Not stableIn-placeFastest in practice

Visual — Partition around pivot 10

def quick_sort(arr):
    a = arr[:]
    _qs(a, 0, len(a) - 1)
    return a

def _qs(a, lo, hi):
    if lo < hi:
        p = _partition(a, lo, hi)
        _qs(a, lo, p - 1)
        _qs(a, p + 1, hi)

def _partition(a, lo, hi):
    pivot = a[hi]; i = lo
    for j in range(lo, hi):
        if a[j] < pivot:
            a[i], a[j] = a[j], a[i]; i += 1
    a[i], a[hi] = a[hi], a[i]
    return i

arr = [38, 27, 43, 3, 9, 82, 10]
print(quick_sort(arr))  # [3, 9, 10, 27, 38, 43, 82]

Heap Sort O(n log n)

Uses a binary max-heap. Builds a max-heap from the array (largest at root), then repeatedly extracts the root and places it at the end, shrinking the heap by one each time.

How it works

  1. Build max-heap (heapify bottom-up from last non-leaf).
  2. Root is the maximum element.
  3. Swap root with last element; reduce heap size by 1.
  4. Heapify root to restore max-heap property.
  5. Repeat until heap size is 1.
Best
O(n log n)
Average
O(n log n)
Worst
O(n log n)
Space
O(1)
Not stableIn-placeGuaranteed O(n log n) + O(1) space

Visual — Max-heap tree

def heap_sort(arr):
    a = arr[:]; n = len(a)
    for i in range(n//2-1, -1, -1):
        _heapify(a, n, i)
    for i in range(n-1, 0, -1):
        a[0], a[i] = a[i], a[0]
        _heapify(a, i, 0)
    return a

def _heapify(a, size, root):
    largest = root
    l, r = 2*root+1, 2*root+2
    if l < size and a[l] > a[largest]: largest = l
    if r < size and a[r] > a[largest]: largest = r
    if largest != root:
        a[root], a[largest] = a[largest], a[root]
        _heapify(a, size, largest)

arr = [38, 27, 43, 3, 9, 82, 10]
print(heap_sort(arr))  # [3, 9, 10, 27, 38, 43, 82]

Bucket Sort O(n + k)

A distribution-based algorithm that distributes elements into "buckets" based on value range, sorts each bucket individually, then concatenates them. Best when data is uniformly distributed.

How it works

  1. Determine the range (min and max) of input values.
  2. Create a fixed number of empty buckets spanning that range.
  3. Distribute each element into its corresponding bucket.
  4. Sort each individual bucket (typically with insertion sort).
  5. Concatenate all sorted buckets to get the final result.
Best
O(n + k)
Average
O(n + k)
Worst
O(n²)
Space
O(n + k)
StableNot in-placeBest for uniform distributionNon-comparison sort

Visual — distributing into 4 buckets

def bucket_sort(arr, num_buckets=5):
    if not arr: return arr[:]
    a = arr[:]
    min_val, max_val = min(a), max(a)
    if min_val == max_val: return a

    bucket_range = (max_val - min_val + 1) / num_buckets
    buckets = [[] for _ in range(num_buckets)]

    for val in a:
        idx = int((val - min_val) / bucket_range)
        if idx == num_buckets: idx -= 1
        buckets[idx].append(val)

    for bucket in buckets:
        bucket.sort()

    result = []
    for bucket in buckets:
        result.extend(bucket)
    return result

arr = [38, 27, 43, 3, 9, 82, 10]
print(bucket_sort(arr))  # [3, 9, 10, 27, 38, 43, 82]

When to use what

  1. Small / nearly sorted data → Insertion Sort (adaptive O(n) best)
  2. Stability required → Merge Sort or Insertion Sort
  3. General purpose, fastest in practice → Quick Sort (randomized pivot)
  4. Guaranteed O(n log n) + O(1) space → Heap Sort
  5. Uniform data over known range → Bucket Sort
  6. Linked lists → Merge Sort (no random access needed)
  7. Learning / interviews → Know all seven and their trade-offs
