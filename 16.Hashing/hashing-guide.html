Chapter 01

What is hashing?

Hashing converts data of any size into a fixed-size integer — a fingerprint for your data. That integer maps directly to an array index, so you can store and retrieve in O(1) average time.

Core concept: key → hash → index → bucket
python
# Python's hash() returns an integer for any hashable object
print(hash("apple"))    # e.g. 2572080938546383898
print(hash("banana"))   # e.g. -1649683334505052884
print(hash(42))         # 42  (small ints hash to themselves!)
print(hash((1,2,3)))    # tuples are hashable

# To map hash → array index, use modulo
table_size = 7
index = hash("apple") % table_size   # gives 0-6
print(f'"apple" goes to index {index}')
Chapter 02

Hash functions deep dive

A hash function h(key) maps a key from the universe U to the range [0, m-1] where m is the table size. A good hash function is (1) deterministic, (2) uniformly distributed, (3) has an avalanche effect — a small change in input → large change in output, and (4) is fast to compute.

The division method

Division method formula
h(k) = k mod m
Simplest method. Choose m as a prime not close to a power of 2. If m = 2p, only the lowest p bits matter — bad distribution. Good sizes: 7, 13, 31, 61, 127, 251, 509, 1021.
python — division method
def hash_division(key, m):
    """h(k) = k mod m  —  m should be prime."""
    return key % m

# Example: key=42, m=7
print(hash_division(42, 7))   # 42 % 7 = 0
print(hash_division(15, 7))   # 15 % 7 = 1
print(hash_division(99, 7))   # 99 % 7 = 1  ← collision with 15!

The multiplication method

Multiplication method formula
h(k) = ⌊ m × (k × A mod 1) ⌋
Knuth suggests A ≈ (√5 − 1)/2 ≈ 0.6180339887 (golden ratio conjugate). Advantage: m does not need to be prime — works with any table size, even powers of 2.
python — multiplication method
import math

def hash_multiplication(key, m):
    """h(k) = floor(m * (k * A mod 1))"""
    A = 0.6180339887   # (√5 - 1) / 2
    return math.floor(m * ((key * A) % 1))

# Example: key=123456, m=16
print(hash_multiplication(123456, 16))  # Distributes well even with m=2^4

Polynomial rolling hash (for strings)

Polynomial hash formula
h(s) = ( s[0]×pn-1 + s[1]×pn-2 + … + s[n-1] ) mod m
Each character is weighted by a power of prime base p (typically 31 or 37). Order-dependent: "abc" ≠ "cba". Foundation of the Rabin-Karp string matching algorithm.
python — polynomial hash
def hash_string(s, m, p=31):
    """Polynomial rolling hash for strings."""
    h = 0
    p_pow = 1
    for ch in s:
        h = (h + (ord(ch) - ord('a') + 1) * p_pow) % m
        p_pow = (p_pow * p) % m
    return h

# "abc" → 1×1 + 2×31 + 3×961 = 1 + 62 + 2883 = 2946
print(hash_string("abc", 1000003))    # 2946
print(hash_string("cba", 1000003))    # Different! Order matters

Universal hashing

Universal hash family formula
ha,b(k) = ((a × k + b) mod p) mod m
Choose a ∈ {1, …, p-1} and b ∈ {0, …, p-1} randomly, where p is prime > any key. Collision probability between any two distinct keys ≤ 1/m. Protects against adversarial inputs.
python — universal hashing
import random

def universal_hash(key, m, p=104729):
    """h(k) = ((a*k + b) mod p) mod m"""
    a = random.randint(1, p - 1)
    b = random.randint(0, p - 1)
    return ((a * key + b) % p) % m

Properties of a good hash function: (1) Deterministic — same input always gives same output. (2) Uniform — spreads keys evenly across all slots. (3) Avalanche effect — 1-bit change in input flips ~50% of output bits. (4) Fast — O(1) for fixed-size keys, O(k) for strings of length k.

Chapter 03

Building a hash table from scratch

A hash table is an array of m slots (buckets), indexed 0 to m-1. Every operation follows the same pattern: compute hash → go to bucket → insert/search/delete.

Operations flow: all go through hash → bucket
python — basic hash table (no collision handling)
class HashTable:
    def __init__(self, size=7):
        self.size = size
        self.table = [None] * size   # Fixed-size array of buckets
        self.count = 0

    def _hash(self, key):
        """h(k) = hash(k) mod m"""
        return hash(key) % self.size

    def insert(self, key, value):
        index = self._hash(key)
        self.table[index] = (key, value)   # ⚠️ Overwrites on collision!
        self.count += 1

    def search(self, key):
        index = self._hash(key)
        entry = self.table[index]
        return entry[1] if entry and entry[0] == key else None

    def delete(self, key):
        index = self._hash(key)
        if self.table[index] and self.table[index][0] == key:
            self.table[index] = None
            self.count -= 1

# Usage
ht = HashTable()
ht.insert("apple", 1.50)
ht.insert("banana", 0.75)
print(ht.search("apple"))    # 1.50
print(ht.search("grape"))    # None

Critical flaw: What happens when hash("apple") % 7 == hash("grape") % 7? Both keys want the same slot. That's called a collision — and handling it correctly is the most important concept in hashing.

Chapter 04

Collision resolution

Two families of strategies: separate chaining (linked lists in each bucket) and open addressing (probe for the next empty slot inside the array itself).

Strategy 1 — Separate chaining

Each bucket holds a list. Colliding keys append to the same list. Search scans the list at the target bucket.

Separate chaining — lists in each bucket
python — separate chaining (full implementation)
class HashTableChaining:
    def __init__(self, size=7):
        self.size = size
        self.table = [[] for _ in range(size)]   # Each bucket is a list
        self.count = 0

    def _hash(self, key):
        return hash(key) % self.size

    def insert(self, key, value):
        idx = self._hash(key)
        for i, (k, v) in enumerate(self.table[idx]):
            if k == key:
                self.table[idx][i] = (key, value)   # Update existing
                return
        self.table[idx].append((key, value))         # Append new
        self.count += 1

    def search(self, key):
        idx = self._hash(key)
        for k, v in self.table[idx]:
            if k == key:
                return v
        return None

    def delete(self, key):
        idx = self._hash(key)
        for i, (k, v) in enumerate(self.table[idx]):
            if k == key:
                self.table[idx].pop(i)
                self.count -= 1
                return True
        return False

    def load_factor(self):
        """α = n/m.  Above 0.75 → time to resize!"""
        return self.count / self.size

Strategy 2 — Open addressing

No linked lists. All keys live inside the array. On collision, probe for the next empty slot using a formula. There are three probing methods — each with different trade-offs.

Chapter 05

Probing formulas (detailed)

In open addressing, the probe sequence h(k, 0), h(k, 1), h(k, 2), … determines where to look when the initial slot is taken. The probe number i starts at 0 and increments until an empty slot is found.

Linear probing
h(k, i) = (h(k) + i) mod m
Steps: +1, +2, +3, …
Cache-friendly (sequential memory access).
Causes primary clustering.
Quadratic probing
h(k, i) = (h(k) + c₁i + c₂i²) mod m
Steps: +1, +4, +9, +16, …
Reduces primary clustering.
Causes secondary clustering. Requires m = prime.
Double hashing
h(k, i) = (h₁(k) + i × h₂(k)) mod m
Step size from 2nd hash function.
Each key has unique probe sequence.
No clustering at all.

Linear probing — step by step

Linear probing formula
h(k, i) = ( h(k) + i ) mod m
k = key, i = probe number (0, 1, 2, …), m = table size.
Probe sequence: h(k), h(k)+1, h(k)+2, … all mod m (wraps around).
Example: h("apple")=3, m=7 → try slots 3, 4, 5, 6, 0, 1, 2.

python — linear probing (full)
class LinearProbingHT:
    def __init__(self, m=7):
        self.m = m
        self.keys = [None] * m
        self.vals = [None] * m

    def _probe(self, key, i):
        """h(k, i) = (h(k) + i) mod m"""
        return (hash(key) % self.m + i) % self.m

    def insert(self, key, val):
        for i in range(self.m):                # Try up to m probes
            idx = self._probe(key, i)           # Apply formula
            if self.keys[idx] is None or self.keys[idx] == key:
                self.keys[idx] = key
                self.vals[idx] = val
                return
        raise Exception("Table full!")

    def search(self, key):
        for i in range(self.m):
            idx = self._probe(key, i)
            if self.keys[idx] is None:
                return None                      # Hit empty slot → not found
            if self.keys[idx] == key:
                return self.vals[idx]
        return None

Quadratic probing — step by step

Quadratic probing formula
h(k, i) = ( h(k) + c₁×i + c₂×i² ) mod m
Common choice: c₁ = 0, c₂ = 1, simplifying to h(k, i) = (h(k) + i²) mod m.
Probe sequence: h(k), h(k)+1, h(k)+4, h(k)+9, h(k)+16, …
m must be prime — otherwise not all slots are visited!
When m is prime and table is < half full, insertion is guaranteed.
python — quadratic probing
class QuadraticProbingHT:
    def __init__(self, m=7):          # m must be prime!
        self.m = m
        self.keys = [None] * m
        self.vals = [None] * m

    def _probe(self, key, i):
        """h(k, i) = (h(k) + i²) mod m"""
        return (hash(key) % self.m + i * i) % self.m

    def insert(self, key, val):
        for i in range(self.m):
            idx = self._probe(key, i)
            if self.keys[idx] is None or self.keys[idx] == key:
                self.keys[idx] = key
                self.vals[idx] = val
                return

# Probe sequence for h(k)=3, m=7:
# i=0: (3+0) % 7 = 3
# i=1: (3+1) % 7 = 4
# i=2: (3+4) % 7 = 0
# i=3: (3+9) % 7 = 5

Double hashing — step by step

Double hashing formula
h(k, i) = ( h₁(k) + i × h₂(k) ) mod m
Two hash functions:
h₁(k) = k mod m — determines starting slot
h₂(k) = 1 + (k mod (m − 1)) — determines step size
The +1 ensures h₂ is never 0 (otherwise we'd loop forever on the starting slot).
Each key gets a unique step size → unique probe sequence → no clustering at all.
python — double hashing
class DoubleHashHT:
    def __init__(self, m=7):
        self.m = m
        self.keys = [None] * m
        self.vals = [None] * m

    def _h1(self, key):
        """Primary hash — starting position."""
        return hash(key) % self.m

    def _h2(self, key):
        """Secondary hash — step size. Never returns 0!"""
        return 1 + (hash(key) % (self.m - 1))

    def _probe(self, key, i):
        """h(k,i) = (h1(k) + i * h2(k)) mod m"""
        return (self._h1(key) + i * self._h2(key)) % self.m

    def insert(self, key, val):
        for i in range(self.m):
            idx = self._probe(key, i)
            if self.keys[idx] is None or self.keys[idx] == key:
                self.keys[idx] = key
                self.vals[idx] = val
                return

# Example: key with h1=3, h2=2, m=7
# i=0: (3 + 0×2) % 7 = 3
# i=1: (3 + 1×2) % 7 = 5
# i=2: (3 + 2×2) % 7 = 0
# i=3: (3 + 3×2) % 7 = 2  ← different from linear!
Chapter 06

The clustering problem

Primary clustering (linear probing): Contiguous occupied blocks attract more keys. A block of size k has probability (k+1)/m of growing on the next insert. Clusters snowball — bigger clusters grow faster. Average probes degrade from O(1) toward O(n).

Secondary clustering (quadratic probing): Keys that hash to the same initial slot follow the exact same probe sequence. Less severe than primary clustering, but measurable when many keys share a slot.

Double hashing solves both: The second hash function gives each key a unique step size. Even keys starting at the same slot diverge immediately. This is the closest to "ideal" uniform probing.

Strategies comparison

StrategyClusteringCache perf.DeletionComplexity
Separate chainingNonePoorSimpleLow
Linear probingPrimaryExcellentTombstonesLow
Quadratic probingSecondaryGoodTombstonesMedium
Double hashingNonePoorTombstonesHigh

Tombstone deletion in open addressing

You can't just set a deleted slot to None — it would break the probe chain! Subsequent searches would stop too early and miss keys that were placed past the deleted slot.

Solution: Tombstones. Mark deleted slots with a sentinel value (e.g., DELETED). During search: skip past tombstones (keep probing). During insert: overwrite tombstones. Over time, tombstones accumulate → periodic rehashing cleans them up.

python — tombstone deletion
DELETED = "__TOMBSTONE__"

class OpenAddressHT:
    def delete(self, key):
        for i in range(self.m):
            idx = self._probe(key, i)
            if self.keys[idx] is None:
                return False            # Key not found
            if self.keys[idx] == key:
                self.keys[idx] = DELETED  # Don't use None!
                self.vals[idx] = None
                return True
        return False

    def search(self, key):
        for i in range(self.m):
            idx = self._probe(key, i)
            if self.keys[idx] is None:
                return None             # Empty slot → stop
            if self.keys[idx] == DELETED:
                continue                # Tombstone → keep probing
            if self.keys[idx] == key:
                return self.vals[idx]
Chapter 07

Load factor & dynamic resizing

Load factor formula
α = n / m
n = number of stored elements, m = number of slots.
Higher α → more collisions → slower operations. Keep α below a threshold.

Thresholds & expected probe counts

StrategyMax αProbes (successful)Probes (unsuccessful)
Chaining1.0+1 + α/2α
Linear probing0.5 – 0.7½(1 + 1/(1−α))½(1 + (1/(1−α))²)
Quadratic probing0.5−ln(1−α) / α1 / (1−α)
Double hashing0.5 – 0.7−ln(1−α) / α1 / (1−α)

At α=0.5 linear probing averages 1.5 probes per lookup. At α=0.9 it averages 5.5. At α=0.99 it's 50. This is why resizing is critical!

Rehashing (dynamic resizing)

Rehashing formula
mnew = next_prime( 2 × mold )
Steps: (1) Allocate new table of size m_new. (2) Recompute h(key) % m_new for every key. (3) Re-insert all keys. (4) Replace old table.
Cost: O(n) per resize. But resizes are rare enough that amortized cost is O(1) per insert.
Python's dict resizes at α = 2/3. Table size is always a power of 2.
python — hash table with automatic resizing
class DynamicHashTable:
    def __init__(self, m=7):
        self.m = m
        self.table = [[] for _ in range(m)]
        self.n = 0

    def _resize(self):
        """Double size + 1 (stays odd → closer to prime)."""
        old = self.table
        self.m = self.m * 2 + 1
        self.table = [[] for _ in range(self.m)]
        self.n = 0
        for bucket in old:
            for k, v in bucket:
                self.insert(k, v)           # Re-hash with new m

    def insert(self, key, val):
        if self.n / self.m >= 0.75:       # α threshold → resize!
            self._resize()
        idx = hash(key) % self.m
        for i, (k, v) in enumerate(self.table[idx]):
            if k == key:
                self.table[idx][i] = (key, val)
                return
        self.table[idx].append((key, val))
        self.n += 1

# The table automatically grows when it gets too full!
ht = DynamicHashTable()
for i in range(100):
    ht.insert(f"key_{i}", i)
print(f"Size: {ht.m}, Items: {ht.n}, α: {ht.n/ht.m:.2f}")
Chapter 08

Python dict & set internals

Both dict and set are hash tables. Understanding CPython's specific implementation helps you reason about performance.

CPython dict probe formula (perturbation-based)
j = ((5 × j) + 1 + perturb) mod 2n
perturb >>= 5   (shift right 5 bits each step)
Table size is always 2n (power of 2). The perturb variable starts as the full hash value and gets right-shifted by 5 each probe. Initially it jumps widely across the table (using high-order hash bits); as perturb decays toward 0, it becomes linear. This gives excellent distribution without needing double hashing. Load threshold: α = 2/3. Preserves insertion order since Python 3.7+.
dict vs set internals

"a" in d                         # Membership check   → O(1) ← SUPERPOWER
d.setdefault("e", 5)            # Insert only if missing
d.pop("a", None)                # Remove and return

Pattern 1 — Frequency counting

python
def first_unique_char(s: str) -> int:
    freq = {}
    for ch in s:
        freq[ch] = freq.get(ch, 0) + 1
    for i, ch in enumerate(s):
        if freq[ch] == 1:
            return i
    return -1

print(first_unique_char("leetcode"))   # 0 (→ 'l')

# Anagram check — even shorter with Counter:
from collections import Counter
def is_anagram(s, t):
    return Counter(s) == Counter(t)

Pattern 2 — Two Sum

python
def two_sum(nums: list, target: int) -> list:
    seen = {}   # value → index
    for i, num in enumerate(nums):
        complement = target - num
        if complement in seen:         # O(1) lookup!
            return [seen[complement], i]
        seen[num] = i                  # Store for future
    return []

print(two_sum([2, 7, 11, 15], 9))  # [0, 1]
i=0, num=2 → complement = 9−2 = 7 → 7 not in {} → seen = {2: 0}
i=1, num=7 → complement = 9−7 = 2 → 2 IS in {2:0} → return [0, 1]

Pattern 3 — Sliding window + hash map

python — longest substring without repeating chars
def length_of_longest_substring(s: str) -> int:
    char_index = {}      # char → last seen index
    left = 0
    max_len = 0
    for right, ch in enumerate(s):
        if ch in char_index and char_index[ch] >= left:
            left = char_index[ch] + 1    # Shrink window
        char_index[ch] = right
        max_len = max(max_len, right - left + 1)
    return max_len

print(length_of_longest_substring("abcabcbb"))   # 3 ("abc")

Pattern 4 — Grouping / bucketing

python — group anagrams
from collections import defaultdict

def group_anagrams(strs: list) -> list:
    groups = defaultdict(list)
    for s in strs:
        key = tuple(sorted(s))      # Sorted chars as hashable key
        groups[key].append(s)
    return list(groups.values())

print(group_anagrams(["eat","tea","tan","ate","nat","bat"]))
# [['eat','tea','ate'], ['tan','nat'], ['bat']]

Pattern 5 — Prefix sum + hash map

python — subarray sum equals K
def subarray_sum(nums: list, k: int) -> int:
    prefix_count = {0: 1}    # prefix_sum → count seen
    current_sum = 0
    result = 0
    for num in nums:
        current_sum += num
        if current_sum - k in prefix_count:
            result += prefix_count[current_sum - k]
        prefix_count[current_sum] = prefix_count.get(current_sum, 0) + 1
    return result

print(subarray_sum([1, 1, 1], 2))   # 2

Pattern 6 — Set for O(1) existence checks

python — longest consecutive sequence
def longest_consecutive(nums: list) -> int:
    num_set = set(nums)
    max_streak = 0
    for num in num_set:
        if num - 1 not in num_set:    # Start of a sequence
            current, streak = num, 1
            while current + 1 in num_set:
                current += 1; streak += 1
            max_streak = max(max_streak, streak)
    return max_streak

print(longest_consecutive([100,4,200,1,3,2]))  # 4 (1→2→3→4)

# Contains duplicate — one-liner:
def contains_duplicate(nums):
    return len(nums) != len(set(nums))

Advanced techniques

Custom hashable objects

python — __hash__ and __eq__
class Point:
    def __init__(self, x, y):
        self.x = x; self.y = y
    def __hash__(self):
        return hash((self.x, self.y))    # Delegate to tuple
    def __eq__(self, other):
        return isinstance(other, Point) and self.x==other.x and self.y==other.y

unique = {Point(1,2), Point(1,2), Point(3,4)}
print(len(unique))   # 2 — duplicates removed!

frozenset for unordered cache keys

python
cache = {}
def expensive(items):
    key = frozenset(items)       # Order doesn't matter
    if key in cache: return cache[key]
    result = sum(x**2 for x in items)
    cache[key] = result
    return result

expensive([3,1,2])   # Computes
expensive([2,3,1])   # Cache hit! Same frozenset

LRU Cache with OrderedDict

python — LRU cache (classic interview question)
from collections import OrderedDict

class LRUCache:
    def __init__(self, capacity: int):
        self.cache = OrderedDict()
        self.capacity = capacity

    def get(self, key: int) -> int:
        if key not in self.cache:
            return -1
        self.cache.move_to_end(key)       # Mark as recently used
        return self.cache[key]

    def put(self, key: int, value: int):
        if key in self.cache:
            self.cache.move_to_end(key)
        self.cache[key] = value
        if len(self.cache) > self.capacity:
            self.cache.popitem(last=False)  # Evict LRU

lru = LRUCache(2)
lru.put(1, 1); lru.put(2, 2)
print(lru.get(1))    # 1
lru.put(3, 3)         # Evicts key 2
print(lru.get(2))    # -1 (evicted)
Chapter 12

When not to use hashing

Need sorted order? → Use a balanced BST or sorted array. Dict maintains insertion order, not sort order.

Range queries? ("find all keys between 5 and 10") → Hash tables can't do range queries. Use a tree or sorted structure.

Memory is extremely tight? → Hash tables use ~2–3× memory of their contents due to empty slots and load factor headroom.

Need worst-case O(log n)? → Hash tables can degrade to O(n) with adversarial inputs. Balanced trees give consistent O(log n) for every operation.

Hash function formulas

All hash function methods
Division:       h(k) = k mod m
Multiplication: h(k) = ⌊m × (k×A mod 1)⌋
Polynomial:    h(s) = Σ s[i] × p^(n-1-i) mod m
Universal:     h(k) = ((a×k + b) mod p) mod m

Probing formulas

All open addressing probe sequences
Linear:    h(k, i) = (h(k) + i) mod m
Quadratic: h(k, i) = (h(k) + c₁i + c₂i²) mod m
Double:    h(k, i) = (h₁(k) + i × h₂(k)) mod m
CPython:   j = ((5×j) + 1 + perturb) mod 2n; perturb >>= 5

Performance formulas

Load factor & resizing
Load factor:       α = n / m
Resize trigger:    α > threshold (0.5–0.75)
New table size:    m_new ≈ 2 × m_old (prime)
Amortized insert:  O(1) (resize is O(n) but rare)
