01 What is recursion?

A recursive function is a function that calls itself. Every recursive function has exactly two parts:

■ Base case

The simplest version of the problem that can be answered directly, without further recursion. This is your exit condition.

■ Recursive case

Break the problem into a smaller version of itself, call the function on that smaller input, and combine the result.

Python
def factorial(n):
    if n <= 1:        # base case
        return 1
    return n * factorial(n - 1)  # recursive case
Key insight

The recursive leap of faith: assume the function already works correctly for smaller inputs. You only need to handle one level — the base case and one step of reduction.

02 The recursion template

Nearly every recursive function follows this three-part skeleton. Master this shape and you can tackle any recursive problem:

Template
def solve(problem):
    # 1. Base case — smallest version of the problem
    if problem is trivial:
        return trivial_answer

    # 2. Recursive case — shrink the problem
    smaller = break_down(problem)
    sub_result = solve(smaller)

    # 3. Combine — use sub-result to build the answer
    return combine(sub_result)

03 How the call stack works

Each recursive call creates a new stack frame. The function pauses at each level, waiting for the deeper call to return. Once the base case is hit, results propagate back up — this is called unwinding.

04 Classic building blocks

Sum of a list

Time: O(n)Space: O(n)
Python
def recursive_sum(arr):
    if not arr:          # base case: empty list
        return 0
    return arr[0] + recursive_sum(arr[1:])  # first + sum of rest

Fibonacci — naive vs. memoized

Python
# Naive — O(2^n), very slow
def fib(n):
    if n <= 1:
        return n
    return fib(n - 1) + fib(n - 2)

# Memoized — O(n), each subproblem solved once
from functools import lru_cache

@lru_cache(maxsize=None)
def fib_memo(n):
    if n <= 1:
        return n
    return fib_memo(n - 1) + fib_memo(n - 2)

Coral nodes are recomputed — memoization eliminates this waste

05 Recursion on data structures

Recursive data structures — trees, linked lists, graphs — naturally lend themselves to recursive algorithms.

Linked list: reverse

Time: O(n)Space: O(n)
Python
class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next

def reverse(head):
    if not head or not head.next:   # base case: 0 or 1 node
        return head
    new_head = reverse(head.next)   # reverse the rest
    head.next.next = head           # point next node back to me
    head.next = None                # break forward link
    return new_head

Binary tree operations

Time: O(n)Space: O(h) — h = tree height
Python
class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right

# Inorder traversal (left → root → right)
def inorder(root):
    if not root:
        return []
    return inorder(root.left) + [root.val] + inorder(root.right)

# Max depth of a binary tree
def max_depth(root):
    if not root:
        return 0
    return 1 + max(max_depth(root.left), max_depth(root.right))

# Invert (mirror) a binary tree
def invert_tree(root):
    if not root:
        return None
    root.left, root.right = invert_tree(root.right), invert_tree(root.left)
    return root

Leaves return 0, each parent adds 1 + max of children

06 Divide and conquer

Split the problem in half, solve each half recursively, combine results. This powers the fastest sorting and searching algorithms.

Merge sort

Time: O(n log n)Space: O(n)
Python
def merge_sort(arr):
    if len(arr) <= 1:             # base case
        return arr

    mid = len(arr) // 2
    left = merge_sort(arr[:mid])  # sort left half
    right = merge_sort(arr[mid:]) # sort right half
    return merge(left, right)     # combine sorted halves

def merge(left, right):
    result = []
    i = j = 0
    while i < len(left) and j < len(right):
        if left[i] <= right[j]:
            result.append(left[i]); i += 1
        else:
            result.append(right[j]); j += 1
    result.extend(left[i:])
    result.extend(right[j:])
    return result

Binary search (recursive)

Time: O(log n)Space: O(log n)
Python
def binary_search(arr, target, lo=0, hi=None):
    if hi is None:
        hi = len(arr) - 1
    if lo > hi:
        return -1                           # not found

    mid = (lo + hi) // 2
    if arr[mid] == target:
        return mid                          # found
    elif arr[mid] < target:
        return binary_search(arr, target, mid + 1, hi)
    else:
        return binary_search(arr, target, lo, mid - 1)

07 Backtracking

Backtracking is recursion's most powerful pattern for combinatorics and constraint satisfaction. The three-step cycle: choose → explore → un-choose.

Permutations

Python
def permutations(nums):
    result = []

    def backtrack(path, remaining):
        if not remaining:           # base case: used all
            result.append(path[:])
            return
        for i in range(len(remaining)):
            path.append(remaining[i])           # choose
            backtrack(path, remaining[:i] + remaining[i+1:])  # explore
            path.pop()                           # un-choose!

    backtrack([], nums)
    return result

N-Queens

Python
def solve_n_queens(n):
    results = []

    def backtrack(row, cols, diag1, diag2, board):
        if row == n:
            results.append(["".join(r) for r in board])
            return
        for col in range(n):
            if col in cols or (row-col) in diag1 or (row+col) in diag2:
                continue                  # prune
            board[row][col] = "Q"         # choose
            backtrack(row+1, cols|{col}, diag1|{row-col},
                      diag2|{row+col}, board)  # explore
            board[row][col] = "."         # un-choose

    backtrack(0, set(), set(), set(), [["."]*n for _ in range(n)])
    return results

08 Recursion → dynamic programming

Every DP problem starts as a recursive solution. The optimization path:

Climbing stairs — all four stages

Python
# Step 1: Pure recursion — O(2^n)
def climb(n):
    if n <= 1: return 1
    return climb(n - 1) + climb(n - 2)

# Step 2: Memoization (top-down) — O(n)
def climb_memo(n, memo={}):
    if n <= 1: return 1
    if n not in memo:
        memo[n] = climb_memo(n - 1) + climb_memo(n - 2)
    return memo[n]

# Step 3: Tabulation (bottom-up) — O(n), no recursion
def climb_tab(n):
    if n <= 1: return 1
    dp = [0] * (n + 1)
    dp[0] = dp[1] = 1
    for i in range(2, n + 1):
        dp[i] = dp[i-1] + dp[i-2]
    return dp[n]

# Step 4: Space-optimized — O(n) time, O(1) space
def climb_opt(n):
    a, b = 1, 1
    for _ in range(2, n + 1):
        a, b = b, a + b
    return b

09 Converting to iteration

Python has a default recursion limit of ~1000. For deep recursion, convert to iteration using an explicit stack:

Python
import sys
sys.setrecursionlimit(10000)  # use with caution

# Iterative inorder traversal using explicit stack
def inorder_iterative(root):
    result, stack = [], []
    curr = root
    while curr or stack:
        while curr:              # go as far left as possible
            stack.append(curr)
            curr = curr.left
        curr = stack.pop()      # process node
        result.append(curr.val)
        curr = curr.right       # then go right
    return result
Rule of thumb

Linear recursion (one recursive call) converts to a simple loop. Tree recursion (two+ calls) converts to a loop with an explicit stack or queue.

10 Common pitfalls

PitfallSymptomFix
Missing base case RecursionError Always define when to stop first
Wrong base case Wrong answer for small inputs Test with n=0, n=1, n=2
Not shrinking problem Infinite recursion Each call must move toward the base case
Mutating shared state Weird/wrong results Use path[:] copies or undo mutations
Overlapping subproblems Time limit exceeded Add @lru_cache or a memo dict
Python-specific

Python does not optimize tail recursion. The default stack limit is ~1000 frames. For deep recursion, either increase the limit with sys.setrecursionlimit() or convert to iteration.

11 Practice roadmap

Work through these levels in order. At each problem, start with brute-force recursion, then optimize.

Level 1 — Basics
factorial · fibonacci · sum of list · power(x, n) · reverse string
Level 2 — Trees & linked lists
max depth · inorder traversal · invert tree · reverse linked list · merge two sorted lists
Level 3 — Divide & conquer

Level 4 — Backtracking
permutations · subsets · combination sum · N-queens · sudoku solver · word search
Level 5 — Recursion → DP
