Chapter 01

What is a graph?

A graph is a collection of nodes (vertices) connected by edges. Unlike trees, graphs can have cycles, multiple paths between nodes, and don't require a single root. They model real-world relationships everywhere: social networks, road maps, web pages, dependency systems, and more.

A simple undirected graph with 5 vertices and 6 edges
🔵

Vertex (Node)

A fundamental unit that stores data — a person, city, web page, or any entity.

🔗

Edge

A connection between two vertices — a friendship, road, hyperlink, or dependency.

🔢

Degree

The number of edges connected to a vertex. In directed graphs: in-degree + out-degree.

🛤️

Path

A sequence of vertices connected by edges. Its length is the number of edges traversed.

Chapter 02

Types of graphs

Cyclic

Contains at least one cycle — you can return to a node by following edges.

Acyclic (DAG)

No cycles. Directed acyclic graphs are used for scheduling and dependency resolution.

Connected

A path exists between every pair of vertices — no vertex is isolated.

Disconnected

At least one vertex is unreachable from another — the graph has multiple components.

Chapter 03

Graph representations

There are two standard ways to store a graph in memory. The adjacency list is the default choice in Python — it's memory-efficient for sparse graphs (which most real-world graphs are). The adjacency matrix shines when you need constant-time edge-existence checks.

OperationAdjacency ListAdjacency Matrix
SpaceO(V + E) efficientO(V²) heavy
Add edgeO(1)O(1)
Check edge existsO(degree) scanO(1) instant
Get all neighborsO(degree) fastO(V) scan row
Remove edgeO(degree)O(1)

Rule of thumb: Use an adjacency list (Python dict) for almost everything. Only reach for a matrix when the graph is dense (edges ≈ V²) or you need O(1) edge checks.

Chapter 04

Building a graph class

The foundation. This class stores an adjacency list in a dictionary and supports both directed and undirected, weighted and unweighted graphs.

01_graph_class.pyPython
class Graph:
    """Adjacency list graph — directed or undirected, weighted or unweighted."""

    def __init__(self, directed=False):
        self.adj_list = {}       # {vertex: [(neighbor, weight), ...]}
        self.directed = directed

    def add_vertex(self, vertex):
        if vertex not in self.adj_list:
            self.adj_list[vertex] = []

    def add_edge(self, v1, v2, weight=1):
        self.add_vertex(v1)
        self.add_vertex(v2)
        self.adj_list[v1].append((v2, weight))
        if not self.directed:
            self.adj_list[v2].append((v1, weight))

    def remove_edge(self, v1, v2):
        self.adj_list[v1] = [(n, w) for n, w in self.adj_list[v1] if n != v2]
        if not self.directed:
            self.adj_list[v2] = [(n, w) for n, w in self.adj_list[v2] if n != v1]

    def remove_vertex(self, vertex):
        for v in self.adj_list:
            self.adj_list[v] = [(n, w) for n, w in self.adj_list[v] if n != vertex]
        del self.adj_list[vertex]

    def get_neighbors(self, vertex):
        return [n for n, w in self.adj_list.get(vertex, [])]

    def display(self):
        for vertex, edges in sorted(self.adj_list.items(), key=str):
            neighbors = ', '.join(
                f"{n}(w={w})" if w != 1 else str(n) for n, w in edges
            )
            print(f"  {vertex} → [{neighbors}]")
Chapter 05

BFS — breadth-first search

BFS explores level by level using a queue (FIFO). It visits all neighbors of the current node before moving deeper. This makes it perfect for finding the shortest path in unweighted graphs.

BFS explores A → B, C → D, E, F, G (level by level)
02_bfs.pyPython
from collections import deque

def bfs(graph, start):
    """Traverse level by level using a queue. Time: O(V+E)"""
    visited = set([start])
    queue = deque([start])
    order = []

    while queue:
        vertex = queue.popleft()       # FIFO — take from front
        order.append(vertex)

        for neighbor in graph[vertex]:
            if neighbor not in visited:
                visited.add(neighbor)
                queue.append(neighbor)  # add to the back

    return order
Chapter 06

DFS — depth-first search

DFS dives as deep as possible before backtracking, using a stack (LIFO). It's the backbone of cycle detection, topological sort, and pathfinding in mazes.

03_dfs.pyPython
def dfs_iterative(graph, start):
    """Dive deep using a stack. Time: O(V+E)"""
    visited = set()
    stack = [start]
    order = []

    while stack:
        vertex = stack.pop()           # LIFO — take from top
        if vertex not in visited:
            visited.add(vertex)
            order.append(vertex)
            for neighbor in reversed(graph[vertex]):
                if neighbor not in visited:
                    stack.append(neighbor)
    return order


def dfs_recursive(graph, vertex, visited=None):
    """Elegant recursive version — risks stack overflow on huge graphs."""
    if visited is None:
        visited = set()
    visited.add(vertex)
    print(vertex, end=" ")
    for neighbor in graph[vertex]:
        if neighbor not in visited:
            dfs_recursive(graph, neighbor, visited)
Chapter 07

Shortest path (unweighted)

In an unweighted graph, BFS naturally finds the shortest path because it explores nodes level by level — the first time it reaches a node is via the shortest route.

04_shortest_path_bfs.pyPython
from collections import deque

def shortest_path_bfs(graph, start, end):
    """Find shortest path in an unweighted graph. Time: O(V+E)"""
    if start == end:
        return [start]

    visited = set([start])
    queue = deque([(start, [start])])  # (current_node, path_so_far)

    while queue:
        vertex, path = queue.popleft()

        for neighbor in graph[vertex]:
            if neighbor == end:
                return path + [neighbor]  # found!
            if neighbor not in visited:
                visited.add(neighbor)
                queue.append((neighbor, path + [neighbor]))

    return None  # no path exists
Chapter 08

Dijkstra's algorithm

For weighted graphs with non-negative weights, Dijkstra's algorithm finds the shortest path from a source to all other vertices. It uses a min-heap (priority queue) to always process the closest unvisited vertex next.

Key insight: Dijkstra is essentially BFS with a priority queue instead of a regular queue. Instead of exploring by level, it explores by cumulative distance.

05_dijkstra.pyPython
import heapq

def dijkstra(graph, start):
    """
    Shortest distances from start to all vertices (non-negative weights).
    graph format: {vertex: [(neighbor, weight), ...]}
    Time: O((V + E) log V)
    """
    distances = {v: float('inf') for v in graph}
    distances[start] = 0
    previous = {v: None for v in graph}
    min_heap = [(0, start)]  # (distance, vertex)

    while min_heap:
        current_dist, current = heapq.heappop(min_heap)

        if current_dist > distances[current]:
            continue  # skip — already found a better path

        for neighbor, weight in graph[current]:
            distance = current_dist + weight
            if distance < distances[neighbor]:
                distances[neighbor] = distance
                previous[neighbor] = current
                heapq.heappush(min_heap, (distance, neighbor))

    return distances, previous


def reconstruct_path(previous, start, end):
    """Trace back from end to start using the previous dict."""
    path = []
    current = end
    while current is not None:
        path.append(current)
        current = previous[current]
    path.reverse()
    return path if path[0] == start else None
Chapter 09

Cycle detection

Uses DFS with a three-color scheme: WHITE (unvisited), GRAY (in the current path), and BLACK (fully processed). If we encounter a GRAY node during traversal, we've found a back edge — and therefore a cycle.

06_cycle_detection.pyPython
def has_cycle(graph, directed=False):
    """
    Detect cycles using DFS coloring.
    WHITE=0 (unvisited), GRAY=1 (in-progress), BLACK=2 (done).
    Time: O(V + E)
    """
    WHITE, GRAY, BLACK = 0, 1, 2
    color = {v: WHITE for v in graph}

    def _dfs(vertex, parent):
        color[vertex] = GRAY

        for neighbor in graph[vertex]:
            if color[neighbor] == GRAY:
                # For undirected: skip the edge back to parent
                if directed or neighbor != parent:
                    return True
            if color[neighbor] == WHITE:
                if _dfs(neighbor, vertex):
                    return True

        color[vertex] = BLACK
        return False

    for vertex in graph:
        if color[vertex] == WHITE:
            if _dfs(vertex, None):
                return True
    return False
Chapter 10

Topological sort

Orders vertices in a directed acyclic graph (DAG) so that for every edge u→v, u appears before v. Think of it as scheduling tasks with prerequisites — you can't take Machine Learning before finishing Math and Stats.

Real-world use: Build systems (Makefile), course prerequisites, package dependency resolution (pip, npm), spreadsheet cell evaluation order.

07_topological_sort.pyPython
from collections import deque

def topological_sort(graph):
    """
    Kahn's algorithm (BFS-based topological sort).
    graph format: {vertex: [neighbor, ...]}  (directed)
    Time: O(V + E)
    """
    # Step 1: Calculate in-degrees
    in_degree = {v: 0 for v in graph}
    for v in graph:
        for neighbor in graph[v]:
            in_degree[neighbor] += 1

    # Step 2: Start with nodes that have no prerequisites
    queue = deque([v for v in in_degree if in_degree[v] == 0])
    order = []

    # Step 3: Process
    while queue:
        vertex = queue.popleft()
        order.append(vertex)

        for neighbor in graph[vertex]:
            in_degree[neighbor] -= 1
            if in_degree[neighbor] == 0:
                queue.append(neighbor)

    if len(order) != len(graph):
        raise ValueError("Graph has a cycle!")

    return order
Chapter 11

Connected components

Finds all groups of vertices that are connected to each other. Each component is a "island" in the graph where every vertex can reach every other vertex within the group.

08_connected_components.pyPython
def connected_components(graph):
    """
    Find all connected components in an undirected graph.
    Returns a list of sets. Time: O(V + E)
    """
    visited = set()
    components = []

    for vertex in graph:
        if vertex not in visited:
            component = set()
            stack = [vertex]

            while stack:
                v = stack.pop()
                if v not in visited:
                    visited.add(v)
                    component.add(v)
                    for neighbor in graph[v]:
                        if neighbor not in visited:
                            stack.append(neighbor)

            components.append(component)

    return components
